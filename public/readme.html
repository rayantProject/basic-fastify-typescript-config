🚀Astro baby: Fastify TypeScript Starter

A modern, lightweight setup for a Fastify project with TypeScript, optimized for fast development and efficient builds using esbuild.

📦 Tech Stack

  • Fastify v5.4.0 – Ultra-fast HTTP framework with static file serving
  • TypeScript v5.8.3 – Static typing for Node.js
  • esbuild v0.25.6 – Lightning-fast bundler
  • nodemon, wait-on, concurrently – Hot reloading
  • ESLint v9.31.0 – Code linting
  • Typebox v0.34.37 – Typed schema validation
  • dotenv v17.2.0 – Env variable management

📁 Project Structure

.
├── src/
│   ├── index.ts         # Fastify application entry point
│   ├── server.ts        # Server configuration
│   ├── config/          # Configuration files
│   ├── plugins/         # Fastify plugins
│   └── routes/          # Route handlers
├── dist/                # Compiled files (esbuild output)
├── public/              # Static assets
├── settings/            # Build and project settings
├── esbuild.config.ts    # esbuild configuration
├── eslint.config.mjs    # ESLint configuration
├── tsconfig.json        # TypeScript configuration
├── nodemon.json         # Nodemon configuration
├── package.json         # Dependencies and scripts
└── .env                 # Env variables (optional)

⚙️ Scripts

Command Description
yarn build:tsc Compile with TypeScript
yarn build:esbuild Compile with esbuild
yarn dev Development mode with auto-reload
yarn start Run the built app (dist/index.js)
yarn lint Lint code with ESLint

🚀 Getting Started

Prerequisites

  • Node.js (>= 18.x)
  • Yarn (4.9.2)

Installation

yarn install

Development

yarn dev

Compiles with esbuild, watches files, runs app via nodemon

Production

yarn build:esbuild && yarn start

🔧 Configuration

  • dotenv: Loads env variables from .env
  • ESLint: Flat config with `@eslint/js` & `typescript-eslint`
  • Typebox: Enables schema validation

🛠️ Customization

  • Edit `esbuild.config.ts` for build options
  • Adjust `tsconfig.json` for TS settings
  • Add custom Fastify plugins in `plugins/`

📝 Notes

  • esbuild handles dev + prod builds
  • pino-pretty used for clean logs in dev
  • Extendable ESLint config (JSON, Markdown, etc.)
  • Fastify externals excluded for smaller builds
